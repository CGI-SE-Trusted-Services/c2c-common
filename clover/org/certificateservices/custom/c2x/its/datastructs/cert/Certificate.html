37    * This structure defines how to encode a certificate.
38    * <li> version specifies this certificate's version and shall be set to 1 or 2 depending on version.
39    * <li> Information on this certificate's signer is given in the variable-length vector signer_info.
40    * <li> subject_info specifies information on this certificate's subject. Further information on the subject is
41    * given in the variable-length vector subject_attributes. The elements in the subject_attributes array shall be
42    * encoded in ascending numerical order of their type value, unless this is specifically overridden by a security profile.
43    * subject_attributes shall not contain two entries with the same type value.
44    * <li> The variable-length vector validity_restrictions specifies restrictions regarding this certificate's validity.
45    * The elements in the validity_restrictions array shall be encoded in ascending numerical order of their type value,
46    * unless this is specifically overridden by a security profile. validity_restrictions shall not contain two entries with the same type value.
47    * <li> signature holds the signature of this certificate signed by the responsible CA. The signature shall be calculated over the
48    * encoding of all preceding fields, including all encoded lengths. If the subject_attributes field contains a field of type
49    * reconstruction_value, the signature field shall be omitted.
50    * <p>
51    * NOTE 1: A certificate is considered valid if the current time is within the validity period specified in the certificate, the current region
52    * is within the validity region specified in the certificate, the type of the certificate is valid for the current type of communication,
53    * the signature, which covers all fields except the signature itself, is valid, and the certificate of the signer is valid as signer for the
54    * given certificate's type. If the certificate is self-signed, it is valid if it is stored as a trusted certificate.
55    *
56    * @author Philip Vendil, p.vendil@cgi.com
57    *
58    */
