27    /**
28    * This structure defines how to encode a certificate.
29    * <li> version specifies this certificate's version and shall be set to 1 for conformance with the present document.
30    * <li> Information on this certificate's signer is given in the variable-length vector signer_info.
31    * <li> subject_info specifies information on this certificate's subject. Further information on the subject is
32    * given in the variable-length vector subject_attributes. The elements in the subject_attributes array shall be
33    * encoded in ascending numerical order of their type value, unless this is specifically overridden by a security profile.
34    * subject_attributes shall not contain two entries with the same type value.
35    * <li> The variable-length vector validity_restrictions specifies restrictions regarding this certificate's validity.
36    * The elements in the validity_restrictions array shall be encoded in ascending numerical order of their type value,
37    * unless this is specifically overridden by a security profile. validity_restrictions shall not contain two entries with the same type value.
38    * <li> signature holds the signature of this certificate signed by the responsible CA. The signature shall be calculated over the
39    * encoding of all preceding fields, including all encoded lengths. If the subject_attributes field contains a field of type
40    * reconstruction_value, the signature field shall be omitted.
41    * <p>
42    * NOTE 1: A certificate is considered valid if the current time is within the validity period specified in the certificate, the current region
43    * is within the validity region specified in the certificate, the type of the certificate is valid for the current type of communication,
44    * the signature, which covers all fields except the signature itself, is valid, and the certificate of the signer is valid as signer for the
45    * given certificate's type. If the certificate is self-signed, it is valid if it is stored as a trusted certificate.
46    *
47    * @author Philip Vendil, p.vendil@cgi.com
48    *
49    */
